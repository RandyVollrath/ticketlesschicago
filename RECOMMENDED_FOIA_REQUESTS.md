# Recommended FOIA Requests for Ticketless Chicago

## Overview
These FOIA requests would significantly improve the ticket contest system by providing data to help users make informed decisions and build stronger cases.

## üö¶ Red Light Camera Data

### 1. Camera Calibration and Maintenance Records
**Request to:** Chicago Department of Transportation (CDOT)

**Request text:**
```
I am requesting records for all red light camera intersections in Chicago for the past 3 years:
1. Calibration certificates and test results for all cameras
2. Maintenance logs including repairs, replacements, and downtime
3. Failed calibration reports
4. Vendor service records
5. Camera serial numbers and installation dates

Format: Excel/CSV preferred with fields:
- Camera ID / Intersection location
- Date of calibration/maintenance
- Pass/Fail status
- Issues found
- Resolution
```

**Why this helps:**
- Identify cameras with calibration problems
- Build database of "problem intersections"
- When user gets ticket, instantly check if that camera had issues
- Strengthen contests with specific calibration failures

### 2. Yellow Light Timing Data
**Request to:** Chicago Department of Transportation

**Request text:**
```
I am requesting for all red light camera intersections:
1. Yellow light duration settings (in seconds)
2. Speed limit at each intersection
3. Date of last yellow light timing audit
4. Any changes to yellow light timing in past 5 years
5. Engineering studies justifying current timing

Federal standard: 3.0-4.0 seconds minimum

Format: Excel/CSV with:
- Intersection location
- Yellow light duration
- Speed limit
- Last audit date
```

**Why this helps:**
- Identify intersections with too-short yellow lights (automatic defense)
- Calculate if timing violates federal standards
- Show users which intersections are "traps"
- Build heatmap of problematic intersections

### 3. Red Light Camera Violation Statistics by Intersection
**Request to:** Chicago Department of Finance

**Request text:**
```
For each red light camera intersection, past 3 years:
1. Total violations issued per month
2. Revenue generated per intersection
3. Violation rates by time of day
4. Before/after statistics (when camera installed)
5. Any intersections removed or relocated

Format: Monthly data by intersection location
```

**Why this helps:**
- Identify "revenue traps" with suspiciously high ticket rates
- Spot timing manipulation (sudden spikes in violations)
- Show users if their intersection is statistically abnormal
- Context for letters: "This camera issues 3x more tickets than average"

### 5. Speed Camera Signage Compliance Audits
**Request to:** Chicago Department of Transportation

**Request text:**
```

1. Signage placement audits (distance from camera)
2. Sign maintenance/replacement records
3. Complaints about missing or damaged signs

Format: Location-based records with photos if available
```

**Why this helps:**
- Identify locations with inadequate signage (strong defense)
- Provide baseline for users to compare their photos
- Systematic signage failures = dismissal

### 6. School and Park Zone Hours Documentation
**Request to:** Chicago Public Schools & Chicago Park District

**Request text:**
```
For all schools and parks with speed cameras:
1. Official school hours and calendar
2. Days school was closed (holidays, snow days, etc.)
3. Park operating hours

Format: School/park name, address, calendar for past 3 years
```

**Why this helps:**
- Verify if tickets issued outside permitted hours (automatic dismissal)
- Cross-reference ticket time with school closures
- Provide instant defense for tickets during summer/holidays

## üíæ Database Schema for FOIA Data

### New Tables Needed:

**camera_calibration_records**
```sql
CREATE TABLE camera_calibration_records (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  camera_id TEXT NOT NULL,
  location TEXT NOT NULL,
  calibration_date DATE NOT NULL,
  pass_fail TEXT NOT NULL,
  issues_found TEXT[],
  resolution TEXT,
  vendor TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_camera_location ON camera_calibration_records(location);
CREATE INDEX idx_calibration_date ON camera_calibration_records(calibration_date);
```

**yellow_light_timing**
```sql
CREATE TABLE yellow_light_timing (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  intersection TEXT NOT NULL,
  yellow_duration_seconds DECIMAL(3,1) NOT NULL,
  speed_limit INTEGER,
  last_audit_date DATE,
  meets_federal_standard BOOLEAN,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_intersection ON yellow_light_timing(intersection);
```

**hearing_officer_stats**
```sql
CREATE TABLE hearing_officer_stats (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  officer_id TEXT NOT NULL,
  month DATE NOT NULL,
  total_cases INTEGER,
  dismissals INTEGER,
  reductions INTEGER,
  dismissal_rate DECIMAL(5,2),
  avg_reduction_amount DECIMAL(8,2),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_officer_month ON hearing_officer_stats(officer_id, month);
```

## üìù Submitting the FOIAs

**Chicago FOIA Portal:**
https://www.chicago.gov/city/en/depts/dol/provdrs/foia.html

**Tips:**
1. Submit online for tracking
2. Be specific but not overly narrow
3. Request electronic format (CSV/Excel)
4. Cite Illinois FOIA 5 ILCS 140/3(c) for public records
5. They have 5 business days to respond (can extend 5 more)
6. If denied, you can appeal

**Cost:**
- First 50 pages free
- Electronic records usually free
- Large databases might require "commercial purpose" fee
- Argue "public interest" exception for fee waiver
