## What would be needed to unblock this agent

One of the following:
1. **Allow the sandbox outbound network** — permit the agent's curl/fetch calls
   to reach `www.autopilotamerica.com` with a Bearer token.
2. **Provide a read-only `.env.local`** in the repo (or as a Vercel env on the
   preview environment) so the agent can run `diagnose()` locally against
   Supabase.
3. **Cache the diagnosis JSON as a repo artifact** — have the daily cron
   (`pages/api/cron/parking-quality-daily.ts`) write its `DiagnosisReport` to a
   file like `docs/parking-quality-diagnoses/latest.json` on each run; the agent
   reads that file instead of calling the HTTP endpoint.

Option 3 is the easiest and avoids secrets entirely.
